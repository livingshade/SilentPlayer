# 产品设计

## 产品定位

这是一个只播放本地音乐的个人播放器，不做流媒体、不做社交、不做推荐流。体验上接近 Apple Music、Spotify、Doppler、Foobar2000 mobile 这类成熟播放器，但重点放在本地曲库的可靠管理和稳定听感。

核心承诺：

- 打开快，扫描清楚。
- 本地音乐文件不上传。
- 不同歌曲之间音量尽量一致。
- macOS 和 iPhone 使用同一个曲库规则、播放队列规则和 normalize 规则。

## 信息架构

主导航建议分为 5 个 tab：

- 曲库：歌曲、专辑、艺人、文件夹、最近添加。
- 播放中：封面、进度、队列、歌词或频谱区域。
- 搜索：本地曲库即时搜索。
- 播放列表：用户创建的本地 playlist。
- 设置：音频、扫描、normalize、存储、关于。

macOS 可以使用侧边栏承载这些入口；iPhone 用底部 tab。

## 主界面

### 曲库

曲库页需要支持：

- 按歌曲、专辑、艺人、文件夹切换。
- 排序：标题、艺人、专辑、时长、添加时间。
- 过滤：格式、是否已分析响度、是否缺少封面。
- 批量操作：加入队列、加入播放列表、重新分析响度。

### 播放中

播放中页需要支持：

- 播放/暂停、上一首、下一首。
- 拖动进度条。
- 循环：关闭、单曲、列表。
- 随机播放。
- 收藏。
- 查看当前 normalize 增益，例如 `+3.1 dB`。
- 进入队列编辑。

### 队列

队列是主流播放器体验里很重要的部分：

- 当前播放项固定高亮。
- 可拖拽排序。
- 可从当前位置后插播。
- 可清空队列。
- 支持“播放下一首”和“加入队尾”。

## Normalize 设置

设置页建议提供：

- 响度归一化：开/关。
- 模式：单曲、专辑、智能。
- 目标响度：安静、标准、响亮，分别对应大约 `-18 LUFS`、`-16 LUFS`、`-14 LUFS`。
- 防削波：默认开启。
- 预放大：高级选项，范围建议 `-6 dB` 到 `+6 dB`。
- 自动分析：导入新歌曲后后台分析。

文案应避免技术门槛，用户看到的是：

- “让不同歌曲音量更一致”
- “保留专辑原本的音量关系”
- “防止过响歌曲失真”

## 本地文件模型

iPhone 上不能像桌面系统一样随便扫描整个文件系统，所以建议支持：

- 从 Files App 选择文件夹。
- 从 Files App 导入单首或多首歌曲。
- 保存安全书签，后续继续访问用户授权过的目录。
- 可选复制到 App 沙盒，便于离线和后台访问。

macOS 上建议支持：

- 选择音乐目录。
- 监听目录变化。
- 可选读取 `~/Music`。

## MVP 范围

第一版应包含：

- 本地目录导入。
- 曲库列表。
- 本地搜索。
- 播放/暂停/上一首/下一首。
- 播放队列。
- 播放列表、收藏和最近播放。
- 内嵌封面读取和缓存。
- 单曲 normalize。
- 后台响度分析状态。
- iOS 锁屏控制和耳机控制。
- macOS 菜单栏基础控制。

暂不做：

- 云同步。
- 在线歌词。
- 在线元数据补全。
- 社交、推荐、账号体系。
